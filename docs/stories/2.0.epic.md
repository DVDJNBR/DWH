## Competency Checklist (C13–C17)

### C13 — Modélisation de la structure des données

- [ ] Les données nécessaires aux analyses sont listées de manière exhaustive
- [ ] Les modélisations logiques et physiques sont explicitées sans erreur
- [ ] Application des pratiques standards : modèle en étoile
- [ ] Justification de l'approche bottom-up

### C14 — Création de l'entrepôt de données

- [ ] L'entrepôt remplit toutes les fonctionnalités attendues
- [ ] Explicitation des configurations principales (Terraform)
- [ ] Configuration correcte des accès (sources + analytiques)
- [ ] Procédure de test couvrant le spectre technique et fonctionnel
- [ ] Documentation technique détaillée (architecture, installation, configuration)
- [ ] Retour d'expérience sur la pile technique

### C15 — Intégration des flux ETL

- [ ] Connaissance des formats et volumes de données sources et cibles
- [ ] Alimentation correcte des ETL
- [ ] Respect du format attendu en sortie
- [ ] Application des traitements de nettoyage (quarantaine)
- [ ] Explicitation des règles de gestion de chaque ETL

### C16 — Gestion et supervision de l'entrepôt

- [ ] Journalisation catégorisée avec notifications (email)
- [ ] Priorisation et assignation des tâches de maintenance (SLA)
- [ ] Tableau de bord de suivi des indicateurs
- [ ] Sauvegardes partielles et complètes fonctionnelles
- [ ] Documentation restructurée par cas d'usage
- [ ] Conformité RGPD : registre des traitements et procédures

### C17 — Implémentation des variations (Historisation)

- [ ] Modélisation intégrant et historisant les changements
- [ ] Intégration des variations respectant la modélisation initiale + MAJ ETL
- [ ] Mise à jour de la documentation
